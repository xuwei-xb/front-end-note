## 一、JavaScript 核心

### 1.1 数据类型

#### 分类

| 类型 | 说明 | 存储位置 |
|------|------|----------|
| **基本类型** | Number、String、Boolean、null、undefined、Symbol、BigInt | 栈内存 |
| **引用类型** | Object、Array、Function、Date、RegExp 等 | 堆内存 |

#### 内存模型

- **栈内存**：存储基本类型值，大小固定，自动分配释放
- **堆内存**：存储引用类型值，大小不固定，栈中存储指向堆的地址指针

#### 类型判断方法

| 方法 | 适用场景 | 局限性 |
|------|----------|--------|
| `typeof` | 基本类型判断 | `null` 返回 `"object"`，数组返回 `"object"` |
| `instanceof` | 引用类型判断 | 无法判断基本类型，跨 iframe 失效 |
| `Object.prototype.toString.call()` | 精确判断所有类型 | 写法较繁琐 |
| `constructor` | 判断构造函数 | 可被修改，不安全 |

```javascript
// 最可靠的类型判断
function getType(value) {
  return Object.prototype.toString.call(value).slice(8, -1);
}
// getType([]) → 'Array'
// getType(null) → 'Null'
```

### 1.2 执行上下文与作用域

#### 执行上下文

执行上下文代表代码执行的环境，包含：
- **变量对象（VO）**：收集变量、函数声明、形参
- **作用域链**：变量查找的链条
- **this 指向**

#### 执行流程

```
全局代码执行前 → 创建全局执行上下文 → 压入执行栈
函数调用时 → 创建函数执行上下文 → 压入执行栈
函数执行完毕 → 弹出执行栈
```

#### 作用域链

函数定义时，内部 `[[Scopes]]` 属性保存上级作用域链；执行时，将当前变量对象添加到链首，形成完整的作用域链。

#### 作用域 vs 执行上下文

| 维度 | 作用域 | 执行上下文 |
|------|--------|-----------|
| 创建时机 | 函数定义时 | 函数调用时 |
| 特性 | 静态，不可变 | 动态，随调用创建销毁 |
| 关系 | 执行上下文从属于作用域 |

### 1.3 闭包

#### 定义

函数嵌套，内部函数引用外部函数的变量，当内部函数被传递到外部调用时形成闭包。

#### 形成条件

1. 函数嵌套
2. 内部函数引用外部函数变量
3. 内部函数被传递到外部执行

#### 经典问题

```javascript
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 100);
}
// 输出：3, 3, 3

// 解决方案 1：let 块级作用域
for (let i = 0; i < 3; i++) { ... }

// 解决方案 2：IIFE 形成闭包
for (var i = 0; i < 3; i++) {
  ((j) => setTimeout(() => console.log(j), 100))(i);
}
```

#### 应用场景

- 数据私有化
- 柯里化函数
- 模块模式

#### 内存泄漏

闭包会延长外部函数变量的生命周期，需注意及时解除引用：

```javascript
function outer() {
  const data = new Array(1000000);
  return function inner() {
    console.log(data.length); // 闭包引用 data
  };
}
const fn = outer();
fn = null; // 解除引用，允许垃圾回收
```

### 1.4 原型与原型链

#### 核心概念

| 属性 | 存在位置 | 作用 |
|------|----------|------|
| `prototype` | 函数对象 | 指向原型对象 |
| `__proto__` | 所有对象（非标准） | 指向构造函数的原型 |
| `constructor` | 原型对象 | 指向构造函数 |

#### 原型链

对象通过 `__proto__` 逐级向上查找属性，直到 `Object.prototype.__proto__ === null`。

```
实例 → 构造函数.prototype → Object.prototype → null
```

#### 继承实现

**寄生组合式继承（最优方案）**：

```javascript
function inherit(Child, Parent) {
  const F = function() {};
  F.prototype = Parent.prototype;
  Child.prototype = new F();
  Child.prototype.constructor = Child;
}
```

**ES6 class 继承**：

```javascript
class Child extends Parent {
  constructor() {
    super(); // 必须调用
  }
}
```

### 1.5 this 指向

#### 绑定规则

| 规则 | 场景 | this 指向 |
|------|------|----------|
| 默认绑定 | 独立函数调用 | 严格模式 `undefined`，非严格模式 `window` |
| 隐式绑定 | 对象方法调用 | 调用对象 |
| 显式绑定 | call/apply/bind | 第一个参数 |
| new 绑定 | 构造函数 | 新创建的实例 |

#### 箭头函数

- 没有 `this`，捕获定义时的外层 `this`
- 不能作为构造函数
- 没有 `arguments`，使用 `...rest` 替代

#### call / apply / bind

| 方法 | 执行方式 | 参数形式 |
|------|----------|----------|
| `call` | 立即执行 | 逐个传入 |
| `apply` | 立即执行 | 数组形式 |
| `bind` | 返回新函数 | 逐个传入 |

```javascript
// bind 实现
Function.prototype.myBind = function(context, ...args1) {
  const fn = this;
  return function(...args2) {
    return fn.apply(context, [...args1, ...args2]);
  };
};
```

### 1.6 事件循环

#### 宏任务与微任务

| 类型 | 示例 |
|------|------|
| **宏任务** | script、setTimeout、setInterval、I/O、UI 渲染 |
| **微任务** | Promise.then、MutationObserver、queueMicrotask |

#### 执行流程

```
1. 执行同步代码（宏任务）
2. 检查微任务队列，依次执行所有微任务
3. UI 渲染
4. 执行下一个宏任务
```

#### 经典题目

```javascript
console.log(1);
setTimeout(() => console.log(2), 0);
Promise.resolve().then(() => console.log(3));
console.log(4);
// 输出：1, 4, 3, 2
```

### 1.8 异步编程

#### Promise

```javascript
// 手写 Promise 核心
class MyPromise {
  constructor(executor) {
    this.state = 'pending';
    this.value = undefined;
    this.callbacks = [];
    
    const resolve = (value) => {
      if (this.state !== 'pending') return;
      this.state = 'fulfilled';
      this.value = value;
      this.callbacks.forEach(cb => cb.onFulfilled(value));
    };
    
    const reject = (reason) => {
      if (this.state !== 'pending') return;
      this.state = 'rejected';
      this.value = reason;
      this.callbacks.forEach(cb => cb.onRejected(reason));
    };
    
    executor(resolve, reject);
  }
  
  then(onFulfilled, onRejected) {
    if (this.state === 'pending') {
      this.callbacks.push({ onFulfilled, onRejected });
    }
    // ... 其他状态处理
  }
}
```

#### async/await

```javascript
async function fetchUser() {
  try {
    const response = await fetch('/api/user');
    const data = await response.json();
    return data;
  } catch (error) {
    console.error(error);
  }
}
```

## 二、浏览器原理

### 2.1 渲染流程

```
HTML 解析 → DOM 树
CSS 解析 → CSSOM 树
      ↓
渲染树（Render Tree）
      ↓
布局（Layout/Reflow）
      ↓
绘制（Paint）
      ↓
合成（Composite）
```

#### 关键步骤

1. **HTML 解析**：构建 DOM 树，遇到 `link` 不阻塞（预解析），遇到 `script` 阻塞
2. **样式计算**：生成 Computed Style，相对单位转绝对值
3. **布局**：计算几何信息，生成布局树
4. **分层**：`transform`、`opacity`、`will-change` 等影响分层
5. **绘制**：生成绘制指令
6. **光栅化**：GPU 将块转为位图
7. **合成**：`transform` 在合成线程执行，不阻塞主线程

### 2.2 阻塞解析

| 资源 | 是否阻塞 HTML 解析 | 说明 |
|------|-------------------|------|
| CSS | 否（预解析下载） | 阻塞渲染，阻塞 JS 执行 |
| JS（无属性） | 是 | 下载 + 执行都阻塞 |
| JS + `defer` | 否 | HTML 解析完成后按顺序执行 |
| JS + `async` | 否 | 下载完成立即执行，不保证顺序 |

### 2.3 回流与重绘

#### 回流（Reflow）

改变元素几何属性触发：尺寸、位置、边距等。

**触发场景**：
- 首次渲染
- 窗口 resize
- 元素尺寸/位置变化
- 字体变化

#### 重绘（Repaint）

改变元素外观触发：颜色、背景、阴影等。

**关系**：回流必重绘，重绘不一定回流。

#### 优化策略

```javascript
// 1. 批量修改样式
el.classList.add('active'); // 代替逐个修改 style

// 2. 离线操作 DOM
const fragment = document.createDocumentFragment();
items.forEach(item => fragment.appendChild(item));
container.appendChild(fragment);

// 3. 读写分离
const height = el.offsetHeight; // 读
el.style.height = height + 10 + 'px'; // 写

// 4. 绝对定位脱离文档流
el.style.position = 'absolute';
```

### 2.4 垃圾回收

#### 标记清除

1. 垃圾收集器标记所有变量
2. 去除环境中的变量和被引用变量的标记
3. 清除带标记的变量

#### 引用计数（已弃用）

记录每个值被引用的次数，循环引用导致内存泄漏。

## 三、CSS 布局与优化

### 3.1 BFC（块级格式化上下文）

#### 触发条件

- `float` 不为 `none`
- `position` 为 `absolute` 或 `fixed`
- `display` 为 `inline-block`、`table-cell`、`flex`、`grid`
- `overflow` 不为 `visible`

#### 特性

1. 内部元素垂直排列
2. 不与浮动元素重叠
3. 计算高度时包含浮动元素
4. 独立容器，内外互不影响

#### 应用场景

- 清除浮动
- 阻止 margin 重叠
- 自适应两栏布局

---

### 3.2 元素隐藏

| 方式 | 占据空间 | 交互事件 | 性能 |
|------|----------|----------|------|
| `display: none` | 否 | 否 | 触发回流 |
| `visibility: hidden` | 是 | 否 | 触发重绘 |
| `opacity: 0` | 是 | 是 | 触发重绘 |
| `position: absolute` 移出视口 | 否 | 是 | 无影响 |

---

### 3.3 Flex 布局

```css
.container {
  display: flex;
  justify-content: center; /* 主轴对齐 */
  align-items: center;     /* 交叉轴对齐 */
  flex-wrap: wrap;         /* 换行 */
}

.item {
  flex: 1; /* flex-grow: 1; flex-shrink: 1; flex-basis: 0%; */
}
```

### 3.4 CSS 优化

1. **减少嵌套**：不超过三层
2. **避免通配符**：`*` 性能差
3. **利用继承**：父元素设置可继承属性
4. **CSS Sprite**：合并小图标
5. **避免昂贵属性**：`box-shadow`、`filter`、`:nth-child`
6. **不使用 `@import`**：阻塞并行下载
7. **压缩与 Gzip**

## 五、Vue 框架体系

### 5.1 Vue 2 vs Vue 3 对比

| 维度 | Vue 2 | Vue 3 |
|------|-------|-------|
| 响应式 | `Object.defineProperty` | `Proxy` |
| API 风格 | Options API | Composition API |
| 生命周期 | `beforeCreate` 等 | `setup()`、`onMounted` 等 |
| 根节点 | 单一根节点 | 支持多根节点（Fragment） |
| TypeScript | 支持有限 | 全面支持 |
| Tree-shaking | 不支持 | 支持 |

### 5.2 响应式原理

#### Vue 2

```javascript
Object.defineProperty(data, 'key', {
  get() { /* 收集依赖 */ },
  set() { /* 触发更新 */ }
});
```

**缺陷**：
- 无法监听新增/删除属性
- 无法监听数组索引和长度变化

#### Vue 3

```javascript
const proxy = new Proxy(target, {
  get(target, key) { /* 收集依赖 */ },
  set(target, key, value) { /* 触发更新 */ },
  deleteProperty(target, key) { /* 触发更新 */ }
});
```

**优势**：监听对象所有操作，支持 Map/Set，惰性递归。

### 5.3 生命周期

#### Vue 2 → Vue 3 映射

| Vue 2 | Vue 3 |
|-------|-------|
| `beforeCreate` | `setup()` |
| `created` | `setup()` |
| `beforeMount` | `onBeforeMount` |
| `mounted` | `onMounted` |
| `beforeUpdate` | `onBeforeUpdate` |
| `updated` | `onUpdated` |
| `beforeDestroy` | `onBeforeUnmount` |
| `destroyed` | `onUnmounted` |

#### 父子组件执行顺序

```
创建：父 beforeCreate → 父 created → 子 beforeCreate → 子 created → 子 mounted → 父 mounted
更新：父 beforeUpdate → 子 beforeUpdate → 子 updated → 父 updated
销毁：父 beforeUnmount → 子 beforeUnmount → 子 unmounted → 父 unmounted
```

### 8.1 HTTP 版本对比

3. 客户端验证证书，生成对称密钥
4. 用公钥加密密钥发送

### 8.5 Web 安全

#### XSS（跨站脚本攻击）

**防御**：
- 使用 HTTPS
- 证书校验
- HSTS（强制 HTTPS）

## 附录：常见手写题

### 防抖与节流

```javascript
// 防抖：延迟执行，重复触发重置计时
function debounce(fn, delay) {
  let timer;
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}

// 节流：固定间隔执行
function throttle(fn, interval) {
  let last = 0;
  return function(...args) {
    const now = Date.now();
    if (now - last >= interval) {
      fn.apply(this, args);
      last = now;
    }
  };
}
```

### 深拷贝

```javascript
function deepClone(obj, cache = new WeakMap()) {
  if (obj === null || typeof obj !== 'object') return obj;
  if (cache.has(obj)) return cache.get(obj);
  
  const clone = Array.isArray(obj) ? [] : {};
  cache.set(obj, clone);
  
  for (const key in obj) {
    if (obj.hasOwnProperty(key)) {
      clone[key] = deepClone(obj[key], cache);
    }
  }
  return clone;
}
```

### 发布订阅模式

```javascript
class EventEmitter {
  constructor() {
    this.events = {};
  }
  
  on(event, fn) {
    if (!this.events[event]) this.events[event] = [];
    this.events[event].push(fn);
  }
  
  emit(event, ...args) {
    if (this.events[event]) {
      this.events[event].forEach(fn => fn(...args));
    }
  }
  
  off(event, fn) {
    if (this.events[event]) {
      this.events[event] = this.events[event].filter(cb => cb !== fn);
    }
  }
}
```
