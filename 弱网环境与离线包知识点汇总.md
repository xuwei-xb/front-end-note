# 前端弱网环境与离线包面试知识点汇总

## 一、基础概念类高频题目

### 1. 什么是PWA？列举3个核心特性
PWA即渐进式Web应用（Progressive Web App），是结合网页和原生应用优势的现代Web应用模式，核心特性包括：
- **离线可用**：通过Service Worker缓存资源，无网络时仍能访问核心功能
- **类原生体验**：可安装到桌面，具备全屏模式、启动动画等原生应用特性
- **后台同步**：离线时的操作可在网络恢复后自动同步到服务器

### 2. Service Worker是什么？为什么是PWA的核心？
Service Worker是浏览器在后台独立运行的脚本，独立于网页主线程。它是PWA的核心原因在于：
- 可拦截网络请求，实现离线缓存策略
- 支持消息推送（不依赖页面打开）
- 实现后台同步功能，确保离线操作不丢失

### 3. HTML5离线存储的原理是什么？
HTML5离线存储基于`.appcache`文件的缓存机制，通过解析清单文件离线存储资源。工作流程：
1. 浏览器发现HTML头部有`manifest`属性，请求`manifest`文件
2. 首次访问时根据文件内容下载并离线存储资源
3. 离线状态下浏览器使用缓存资源展示页面

### 4. Manifest.json文件的作用是什么？至少写出5个关键字段
`manifest.json`定义应用安装到桌面时的表现，关键字段包括：
- `name`：应用全称
- `short_name`：应用简称（桌面显示）
- `start_url`：启动路径
- `icons`：桌面图标配置
- `display`：显示模式（standalone/fullscreen）
- `theme_color`：主题颜色

## 二、实现与应用类高频题目

### 1. 如何使用HTML5离线存储？
1. **配置页面**：在HTML头部添加`manifest`属性
```html
<html manifest="demo.appcache">
```

2. **编写manifest文件**：定义缓存资源清单
```
CACHE MANIFEST
#v0.11
CACHE:
  js/app.js
  css/style.css
NETWORK:
  resourse/logo.png
FALLBACK:
  / /offline.html
```

3. **离线操作**：通过`window.applicationCache`接口进行缓存操作

### 2. 如何注册并使用Service Worker实现离线缓存？
1. **注册Service Worker**
```javascript
if ('serviceWorker' in navigator) {
  navigator.serviceWorker.register('/sw.js')
    .then(() => console.log('Service Worker 注册成功'));
}
```

2. **安装阶段缓存资源**：在`sw.js`中实现
```javascript
self.addEventListener('install', event => {
  event.waitUntil(
    caches.open('v1-static')
      .then(cache => cache.addAll(['/index.html', '/app.css', '/main.js']))
  );
});
```

3. **拦截请求返回缓存**
```javascript
self.addEventListener('fetch', event => {
  event.respondWith(
    caches.match(event.request)
      .then(response => response || fetch(event.request))
  );
});
```

### 3. 如何配置浏览器的HTTP缓存策略？
- **强缓存配置**：使用`Cache-Control`设置缓存有效期
```http
Cache-Control: public, max-age=31536000
```

- **协商缓存配置**：结合`ETag`和`Last-Modified`实现
```http
ETag: "123456"
Last-Modified: Wed, 21 Oct 2025 07:28:00 GMT
```

## 三、缓存策略类高频题目

### 1. 常见的缓存策略有哪些？适用场景分别是什么？
- **CacheFirst（缓存优先）**：适用于不常变化的静态资源（JS/CSS），优先从缓存读取，网络异常时仍能使用
- **NetworkFirst（网络优先）**：适用于频繁更新的内容（如新闻列表），优先请求网络，失败时使用缓存
- **Stale-While-Revalidate**：适用于可容忍短时旧数据的场景（如用户头像），立即返回缓存数据，后台更新缓存

### 2. 如何实现带版本号的静态资源缓存和旧缓存清理？
1. **版本化缓存**：在安装阶段使用带版本号的缓存名称
```javascript
const CACHE_NAME = 'v1-static';
caches.open(CACHE_NAME)
  .then(cache => cache.addAll(['/app.123.css', '/main.456.js']));
```

2. **旧缓存清理**：在激活事件中删除非当前版本的缓存
```javascript
self.addEventListener('activate', event => {
  event.waitUntil(
    caches.keys().then(keys => Promise.all(
      keys.map(key => {
        if (key !== CURRENT_CACHE) return caches.delete(key);
      })
    ))
  );
});
```

## 四、高级进阶类高频题目

### 1. Service Worker的生命周期包含哪些阶段？
1. **注册(register)**：页面加载时注册Service Worker脚本
2. **安装(install)**：首次安装时触发，适合缓存静态资源
3. **等待(waiting)**：新版本Service Worker等待旧版本关闭
4. **激活(activate)**：旧版本被替换时触发，适合清理旧缓存
5. **运行(fetch/sync)**：拦截网络请求和处理后台同步

### 2. 如何实现PWA的后台数据同步？
1. **注册同步事件**
```javascript
navigator.serviceWorker.ready
  .then(reg => reg.sync.register('sync-orders'));
```

2. **Service Worker监听同步事件**
```javascript
self.addEventListener('sync', event => {
  if (event.tag === 'sync-orders') {
    event.waitUntil(sendOfflineOrders());
  }
});
```

### 3. 如何处理离线缓存中的缓存污染问题？
- **版本化缓存名称**：每次更新策略修改缓存名称，避免新旧资源混杂
- **按需缓存**：区分静态资源与动态接口，避免缓存敏感数据
- **响应验证**：仅缓存有效的网络响应（HTTP 200状态码）
- **定期清理**：设置缓存上限，按时间或访问频率淘汰旧资源

### 4. iOS对PWA的支持有哪些限制？
- 无后台同步和推送功能支持
- 存储上限50MB
- 7天无访问可能清除缓存数据
- 部分高级API（如蓝牙）不可用

## 五、实战场景类题目

### 1. 设计一个离线优先的新闻阅读PWA，说明实现思路和技术选型
1. **Shell缓存**：使用Service Worker缓存HTML/CSS/JS等核心静态资源
2. **策略设计**：
   - 首页列表：NetworkFirst策略确保获取最新内容
   - 文章详情：CacheFirst策略配合后台更新确保离线可用
3. **数据存储**：使用IndexedDB存储文章内容，Cache存储图片资源
4. **技术选型**：React/Vue作为前端框架，Workbox简化Service Worker开发

### 2. 用户反馈“安装到桌面”按钮不出现，如何排查？
1. **检查manifest配置**：确保`name`、`short_name`、`start_url`、`icons`字段配置正确
2. **HTTPS验证**：确认网站使用HTTPS协议（localhost除外）
3. **访问频率检查**：安装提示需至少两次非连续访问
4. **事件监听检查**：确认`beforeinstallprompt`事件未被阻止

## 六、Manifest与Service Worker缓存对比

| **维度**         | **Manifest（AppCache）**                            | **Service Worker 缓存**                                 |
|------------------|----------------------------------------------------|-------------------------------------------------------|
| **技术本质**     | HTML5 标准化的声明式缓存机制，基于 `.appcache` 文件清单，浏览器自动处理缓存逻辑。 | 独立于网页主线程的可编程网络代理，开发者通过 JS 代码完全控制缓存策略。 |
| **缓存范围**     | 仅能缓存清单中明确列出的静态资源，无法拦截或缓存动态请求（如 API 数据）。 | 可拦截所有网络请求，支持缓存静态资源、动态数据、甚至第三方 CDN 资源。 |
| **更新机制**     | 需修改 `.appcache` 文件内容（如版本注释）触发更新，一旦清单中任意文件加载失败，整个更新流程回退到旧缓存版本。 | 支持灵活的更新策略：通过文件哈希变化、版本号参数或主动调用 `registration.update()` 触发更新，可实现平滑过渡和增量更新。 |
| **缓存策略**     | 固定强缓存模式，无法区分资源类型实现差异化策略，缓存资源过期后需重新下载全部内容。 | 支持多种自定义缓存策略：<br>1. **CacheFirst**：静态资源优先使用缓存<br>2. **NetworkFirst**：动态数据优先请求网络<br>3. **StaleWhileRevalidate**：返回缓存数据的同时后台更新缓存 |
| **兼容性**       | 早期浏览器（IE10+）支持，但已被 W3C 标记为废弃技术，现代浏览器逐步移除支持。 | 现代主流浏览器（Chrome 42+、Firefox 44+、Edge 17+）支持，是 PWA 技术栈的标准组件。 |
| **高级能力**     | 仅支持基础离线缓存，无后台同步、消息推送等扩展能力。 | 支持后台数据同步（Background Sync）、Web Push 消息推送、多缓存版本管理等高级功能。 |
| **适用场景**     | 简单静态网站的基础离线访问需求，适合快速实现低复杂度离线功能。 | 复杂 Web 应用、PWA 项目的全链路离线解决方案，可处理动态数据缓存和复杂业务逻辑。 |
